Lab 03: Build Your Personal Website

Course: ITWS1100 — Fall 2025

DevOps & Lab Setup

  1. Start this lab by creating a new branch called lab03 in your GitHub repository.
  2. In your iit folder, create a new folder named lab03.
  3. Ensure your lab03 folder includes a README.md file and your Information Architecture (IA) outlines from class.
  4. Open your iit folder in VS Code using GitHub Desktop.
  5. As you make changes, GitHub Desktop will show them in the left panel. Commit and push your changes regularly.
  6. Test your website locally using the Live Preview extension (ms-vscode.live-server).
  7. When ready, deploy your code to your Azure instance or personal web server. Pull your changes down and test at FQDN/iit/index.html.

Lab Instructions

  1. Create a personal Intro to ITWS website and deploy it to your personal web server.
    • Your website will be placed in the iit subdirectory (folder) of your Azure instance. This will be the root of your personal web pages.
    • Refer to the tutorials from Lab 1 to get started if needed.
  2. Structure your site as follows:
    • Have a consistent header, navigation menu, content area, and footer across all pages.
    • All resources (CSS, images, etc.) must be kept in appropriate subfolders—no inline styles or new files in your website's root.
    • Required pages:
      • index.html — Your landing (home) page. Edit your Lab 1 file to reflect your personal "brand".
      • projects.html or labs.html — A main page introducing your labs/projects. This should be in a subfolder as per your IA.
      • Landing page for each lab (e.g., Lab 2 resume), with an introduction and links to your solutions.
  3. Populate your projects/labs page with links to your labs, starting with your resume. Add new landing pages as you complete more labs.
  4. Ensure your code is valid, properly indented, and uses semantic HTML. Set the <title> tag on each page.
  5. Only the following files should be in your iit root: index.html, .htaccess, README.md, and any files from Lab 1.
  6. Security must be enabled for your iit website. Unsecured sites will lose points.
  7. Test your site using the Live Preview extension in VS Code, then deploy and test on your personal website.
  8. You will be graded using the following rubric (creativity is rewarded):
    • Validity: /10
    • Indentation: /10
    • Completeness: /50
    • Quality/Creativity: /30
  9. An HTML5 template is provided to help you get started.
  10. Use the web as a reference! (e.g., W3Schools)

Submission

  1. This lab is due by the date specified in the LMS assignment panel.
  2. Update your README.md file in your root directory with comments about the lab, justifications for your chosen methods/styles/tech, and the correct URL for your site (remove any passwords from Lab 1).
  3. Ensure you have a copy of your iit folder:
    • On your local machine for editing
    • In your GitHub repository
    • On your Azure instance
  4. Create a zip file of your iit repository named yourRCSid-ITWS1100-Lab3.zip and submit it to the Assignments area of LMS. (Include all relevant HTML, CSS, README, and resources.)
  5. Include a working URL to your iit homepage in your README file.
  6. Do NOT delete your branches until you have received your grade for this lab.
Tip: Be creative! This website is your professional presence—make it stand out and reflect your interests and skills.
