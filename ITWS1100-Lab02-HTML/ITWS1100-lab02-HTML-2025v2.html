Lab 02: Build Your Resume in HTML & CSS

Course: ITWS1100 — Fall 2025

DevOps & Lab Setup

  1. In Lab 1, you cloned your GitHub repository to your local machine. This folder should be named iit (or your repo name).
  2. In the iit folder, create a new folder called labs.
  3. Inside labs, create a folder named lab01 (for Lab 1) and lab02 (for this lab).
  4. Download the zip file for Lab 02 from LMS to your downloads (or other) folder. Do not put the zip file inside your iit folder.
  5. Unzip the lab02 files into iit/labs/lab02/.
  6. Move your lab1 folder from last week into labs/ as well.
  7. Open your iit folder in VS Code using GitHub Desktop.
  8. As you make changes, GitHub Desktop will show them in the left panel.
  9. To test your code locally, install the Live Preview extension (ms-vscode.live-server).
  10. Commit your changes in GitHub Desktop and push to your GitHub repository when ready.
  11. To deploy to your web server:
    • Start your Azure instance(s).
    • Pull your changes down to your web server(s).
    • Test your resume at FQDN/iit/labs/lab02.

Lab Instructions

  1. Review the provided HTML resume templates and example:
    • Template 1 - Starter
    • Template 2 - Basic
    • Template 3 - Flexbox
    • Template 4 - Grid
    • Example Resume
  2. Rename Intro ITWS - Lab 2 - HTML - template.html to yourRCSID-FullName-resume.html.
  3. Review the README.md file and follow its guidelines.
  4. Edit your HTML file to create your resume. Enter your own information and make your best effort; you can update and correct details later.
  5. After submitting your lab, you will take your resume to the career center for review—so make it good!

Guidelines

  • Name your file: yourRCSID-FullName-resume.html
  • Keep your code in one document. Your CSS should be embedded or linked externally.
  • Do not use tables for layout. Use semantic HTML and CSS (Flexbox/Grid encouraged).
  • Use proper indentation and valid markup (see HTML/CSS lecture notes).
  • Use online resources for reference:
    • W3Schools
    • HTML Validator
    • CSS Validator
  • You will be graded based on the Lab 2 rubric.

Submission

  1. This lab is due by the date specified in LMS.
  2. Create a README.md file in your lab02 folder with any comments/observations about the lab.
  3. Zip your local repo and submit to LMS before the deadline.
  4. Commit and push your edits to GitHub, and deploy them to your web server (see the README for details).
  5. Take your resume to the career center for review.
Tip: Be creative! This resume will be a template for your job search—make it stand out!
